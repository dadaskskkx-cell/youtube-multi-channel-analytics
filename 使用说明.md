# YouTube 多频道收益采集工具 - 使用说明

## 快速开始（3 步完成）

### 第一步：从 Google 下载凭证文件

1. 打开浏览器，访问：https://console.cloud.google.com/
2. 按照下面"详细教程"中的步骤创建项目和凭证
3. 最后点击"下载 JSON"按钮，文件会自动保存到你的"下载"文件夹
4. **重要：下载后不要改文件名，不要移动文件位置，放在哪里都可以**

### 第二步：在工具中选择凭证文件

1. 双击运行 `YouTube多频道收益工具.exe`
2. 在程序顶部找到"OAuth 凭证"这一行
3. 点击右边的"浏览"按钮
4. 在弹出的文件选择窗口中，找到刚才下载的 JSON 文件（通常在"下载"文件夹）
5. 选中文件，点击"打开"
6. 现在输入框里会显示你选择的文件路径

### 第三步：授权你的 YouTube 频道

1. 点击"授权当前频道"按钮
2. 浏览器会自动打开，显示 Google 登录页面
3. 选择你的 YouTube 频道对应的 Google 账号
4. 如果看到"Google 尚未验证此应用"的警告，点击"继续"（这是正常的）
5. 勾选所有权限，点击"继续"
6. 看到"授权成功"后，关闭浏览器
7. 回到工具，你会看到频道已经添加到列表中

**完成！现在可以点击"批量采集数据"按钮开始采集了。**

## 详细教程：如何获取 Google OAuth 凭证

### 步骤 1：创建 Google Cloud 项目

1. 打开浏览器，访问：https://console.cloud.google.com/
2. 如果是第一次使用，会提示你接受服务条款，点击"同意并继续"
3. 点击页面顶部的项目下拉菜单（显示"选择项目"）
4. 在弹出窗口中，点击右上角"新建项目"按钮
5. 项目名称输入：`YouTube数据采集`（或任何你喜欢的名称）
6. 点击"创建"按钮
7. 等待几秒钟，项目创建完成后会自动跳转

### 步骤 2：启用 YouTube Data API

1. 在左侧菜单中，点击"API 和服务"
2. 点击"库"（或"启用的 API 和服务" > "库"）
3. 在搜索框中输入：`YouTube Data API v3`
4. 点击搜索结果中的"YouTube Data API v3"
5. 点击蓝色的"启用"按钮
6. 等待几秒钟，API 启用完成

### 步骤 3：配置 OAuth 同意屏幕

1. 在左侧菜单中，点击"API 和服务" > "OAuth 同意屏幕"
2. 选择"外部"用户类型（单选按钮）
3. 点击"创建"按钮
4. 填写以下信息：
   - **应用名称**：输入 `YouTube数据采集工具`
   - **用户支持电子邮件**：从下拉菜单选择你的邮箱
   - **开发者联系信息**：输入你的邮箱地址
5. 其他字段可以留空，点击"保存并继续"
6. "作用域"页面：直接点击"保存并继续"（不需要添加）
7. "测试用户"页面：
   - 点击"+ ADD USERS"按钮
   - 输入你的 Google 账号邮箱（就是你 YouTube 频道的邮箱）
   - 点击"添加"
   - 点击"保存并继续"
8. 点击"返回到信息中心"

### 步骤 4：创建 OAuth 客户端 ID

1. 在左侧菜单中，点击"API 和服务" > "凭据"
2. 点击页面顶部的"+ 创建凭据"按钮
3. 在下拉菜单中选择"OAuth 客户端 ID"
4. 应用类型选择：**桌面应用**（重要！）
5. 名称输入：`YouTube采集客户端`
6. 点击"创建"按钮

### 步骤 5：下载凭证文件

1. 创建成功后，会弹出一个对话框，显示"OAuth 客户端已创建"
2. 对话框中显示了客户端 ID 和客户端密钥
3. **点击"下载 JSON"按钮**（重要！）
4. 文件会自动下载到你的"下载"文件夹
5. 文件名类似：`client_secret_xxxxx.apps.googleusercontent.com.json`
6. **不要修改文件名，不要移动文件**
7. 点击"确定"关闭对话框

**恭喜！凭证文件已经准备好了，现在可以使用工具了。**

## 使用工具采集数据

### 步骤 1：启动程序

1. 找到 `YouTube多频道收益工具.exe` 文件
2. 双击运行
3. 程序窗口会打开，显示工具界面

### 步骤 2：选择凭证文件

1. 在程序顶部"配置"区域，找到第一行"OAuth 凭证"
2. 点击这一行最右边的"浏览"按钮
3. 会弹出文件选择窗口
4. 在窗口左侧，点击"下载"（或你保存 JSON 文件的位置）
5. 找到之前下载的 JSON 文件（文件名类似 `client_secret_xxxxx.json`）
6. 点击选中这个文件
7. 点击窗口右下角的"打开"按钮
8. 现在"OAuth 凭证"输入框中会显示完整的文件路径

### 步骤 3：授权第一个频道

1. （可选）在"频道别名"输入框中输入一个名称，比如"我的频道"
2. 点击"授权当前频道"按钮
3. 程序会自动打开你的默认浏览器
4. 浏览器显示 Google 登录页面，选择你的 YouTube 频道对应的 Google 账号
5. 如果看到"Google 尚未验证此应用"的警告页面：
   - 这是正常的，因为你的应用还在测试阶段
   - 点击页面左下角的"继续"链接（或点击"Advanced" > "Go to xxx"）
6. 看到权限请求页面，显示需要访问你的 YouTube 数据
7. 勾选所有权限复选框
8. 点击"继续"或"允许"按钮
9. 浏览器会显示"授权成功，可以关闭此页面"
10. 关闭浏览器标签页
11. 回到工具程序，在下方"已授权频道"列表中会看到你的频道信息

### 步骤 4：添加更多频道（可选）

1. 如果你有多个 YouTube 频道需要采集数据
2. 重复"步骤 3"的操作
3. 每次授权时选择不同的 Google 账号
4. 所有频道都会显示在"已授权频道"列表中

### 步骤 5：采集数据

1. 确认"已授权频道"列表中显示了所有需要采集的频道
2. 点击"批量采集数据"按钮
3. 程序会显示进度条，逐个采集每个频道的数据
4. 日志区域会显示采集进度信息
5. 采集完成后，会弹出提示框显示输出文件位置
6. 数据文件默认保存在桌面，文件名类似 `youtube_revenue_data.xlsx`

### 步骤 6：生成中文报表（可选）

1. 点击"生成中文报表"按钮
2. 程序会将英文字段名转换为中文
3. 生成的中文报表文件名类似 `youtube_revenue_data_cn.xlsx`
4. 文件保存在与原数据文件相同的位置

### 步骤 7：查看结果

1. 点击"打开输出文件"按钮，会自动用 Excel 打开数据文件
2. 或者直接到桌面找到 Excel 文件打开查看

## 常见问题解答

### Q1: 为什么需要自己下载 JSON 文件？
**A:** 因为 Google 要求每个用户使用自己的 OAuth 凭证来访问 YouTube API。这样可以：
- 保护你的数据安全
- 避免多人共用配额导致超限
- 符合 Google 的使用政策

### Q2: JSON 文件必须放在 exe 程序旁边吗？
**A:** 不需要！JSON 文件可以放在电脑的任何位置（下载文件夹、桌面、文档等都可以）。使用时通过"浏览"按钮选择文件即可，程序会记住你选择的路径。

### Q3: 需要重命名 JSON 文件吗？
**A:** 不需要！保持下载时的原始文件名即可，程序会自动识别。

### Q4: 授权时看到"Google 尚未验证此应用"怎么办？
**A:** 这是正常的！因为你的应用还在测试阶段。点击"继续"即可，不影响使用。

### Q5: 可以授权多个 YouTube 频道吗？
**A:** 可以！重复授权步骤，每次选择不同的 Google 账号即可。所有频道会显示在列表中。

### Q6: 授权后的 Token 保存在哪里？
**A:** Token 默认保存在程序目录的 `tokens` 文件夹中。如需更改，可以点击"Token 目录"旁的"浏览"按钮选择其他位置。

### Q7: 如何删除已授权的频道？
**A:** 在"已授权频道"列表中点击选中要删除的频道，然后点击"删除选中频道"按钮。

### Q8: 数据采集失败怎么办？
**A:** 查看程序底部"日志"区域的错误信息，常见原因：
- **Token 过期**：重新授权该频道即可
- **API 配额不足**：等待第二天配额重置，或在 Google Cloud Console 申请增加配额
- **网络问题**：检查网络连接是否正常
- **权限不足**：确认授权时勾选了所有权限

### Q9: 可以多个人共用一个 JSON 文件吗？
**A:** 可以，但不建议。因为 Google API 有每日配额限制，多人共用容易超限。建议每个人创建自己的项目和凭证。

### Q10: 数据文件保存在哪里？
**A:** 默认保存在桌面。如需更改，可以点击"输出文件"旁的"浏览"按钮选择其他位置。
